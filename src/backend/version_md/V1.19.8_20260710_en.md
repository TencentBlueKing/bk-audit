## Version 1.19.8 Release Notes

### New Features

- [ NEW ] Added unified platform-level tool and resource management, including tool publish/unpublish controls, Tool Plaza visibility control, and scene- or system-based resource visibility and authorization sharing.
- [ NEW ] Added scene- and system-based visibility settings for platform-level tools, with support for overriding parameter defaults by visibility scope.
- [ NEW ] Added asset synchronization observability metrics, covering sync health, data volume differences, and anomaly event monitoring.

### Optimizations

- [ OPTIMIZATION ] Updated the keyword text in the Tool Plaza User Profile tool's linked game list from "exchange rate not set" to "token exchange ratio not set" when the token metadata field is empty.
- [ OPTIMIZATION ] Improved strategy list loading to reduce cases where the list is not displayed because the tag API fails or the component is not ready.
- [ OPTIMIZATION ] Improved scene information loading by batching related systems and data tables for better page performance with large datasets.
- [ OPTIMIZATION ] Adjusted resigned user search and tag colors to distinguish resigned users from normal search matches.
- [ OPTIMIZATION ] Improved filtering, display, and operation experience for platform management tool and report lists.

### Fixed

- [ FIXED ] Fixed the issue where handling rule field mappings were not displayed.
- [ FIXED ] Fixed the issue where the "Use default value" checkbox could not be unchecked when editing BKVision tool variables.
- [ FIXED ] Fixed duplicate rendering of BKVision charts.
- [ FIXED ] Fixed an exception when IAMV4 policy field values referenced orphan systems.
