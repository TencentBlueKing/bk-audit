## Version 1.19.10 Release Notes

### Optimizations

- [ OPTIMIZATION ] Improved the risk detail page display and handling experience, including basic information, linked event timeline, risk summary, ticket handling, and event investigation report display.
- [ OPTIMIZATION ] Improved system access entry points, supporting separate paths for new systems and existing registered systems from the system list.
- [ OPTIMIZATION ] Improved API tool configuration by allowing person selector input for inactive users and preserving debug status when only descriptive information is edited.
- [ OPTIMIZATION ] Improved execution forms for API and SQL tools by hiding the query input area when no visible query parameters exist.
- [ OPTIMIZATION ] Improved strategy list loading, filter refresh, and column header display.
- [ OPTIMIZATION ] Improved the visible-scope jump and search experience in the Platform Management tool list.

### Fixed

- [ FIXED ] Fixed missing field type icons in log collection field-cleaning mapping.
- [ FIXED ] Fixed incorrect wording in the tool type column in Platform Management > Tool Management.
- [ FIXED ] Fixed missing visible-parameter UI for Data Query tools and incorrect time-range parameter formatting during tool execution.
- [ FIXED ] Fixed strategy list loading failures and refresh failures after filtering.
- [ FIXED ] Fixed table copy and paste issues in the rich text editor.
- [ FIXED ] Fixed risk detail display issues, including missing linked event titles and duplicated linked-event timeline items.
- [ FIXED ] Fixed regex safety validation issues in some input scenarios.
